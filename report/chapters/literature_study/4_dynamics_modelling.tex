A continuous time dynamical system can at its simplist be considered in the following form:

\begin{equation}
    \diff{\mathbf{x}(t)}{t} = \mathbf{f}\left(\mathbf{x}(t), t; \boldsymbol{\beta}\right)
\end{equation}

A discrete time system can analogously be considered in the following form:

\begin{equation}
    \mathbf{x}_{k + 1} = \mathbf{F}(\mathbf{x}_k, t; \mathbb{\beta})
    \label{eq:discrete_dynamical_system}
\end{equation}

As this only depends on the state at the current time, these processes can be considerd as markov processes.

\section{Operator learning}

\subsection{Koopman operator theory - 3p}
Koopman operator theory states that it is possible to represent a nonlinear dynamical system in terms of an
infinite-dimensioned linear operator acting on a Hilbert space of measurement functions of the state of the system
\parencite{bruntonDatadrivenScienceEngineering2019}. This was demonstrated by
\citeauthor{koopmanHamiltonianSystemsTransformation1931} in \citeyear{koopmanHamiltonianSystemsTransformation1931}. This
operator is linear, making analysis of the system much easier. It is however infinite, so must be truncated for real
applications. 

\subsubsection{Mathematical Formulation}
The Koopman operator is a linear operator that acts on a measurement function as follows:
\begin{equation}
    \mathcal{K} g = g(\mathbf{F})
\end{equation}
For a discrete system as described in \eqref{eq:discrete_dynamical_system}, this formulation becomes the following:

\begin{align}
    \mathcal{K}g(\mathbf{x}_k) = g(\mathbf{F}(\mathbf{x}_k)) \\
    \mathcal{K} g(\mathbf{x}_k) = g(\mathbf{x}_{k + 1})
\end{align}

This means the Koopman Operator is a linear operator that advances the observation of a state forward in time. This is
true for any observable function \( g\) and any state \( \mathbf{x}_k \). \Cref{fig:koopman_visualisation} visualises
the transformation performed by the Koopman operator and the measurement functions.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\linewidth]{report/images/literature_study/bruntonDataDrivenDynamicalSystems2019.png}
    \caption{Schematic illustrating the Koopman operator for nonlinear dynamical systems. The dashed lines from yk → xk
    indicate that we would like to be able to recover the original state.
    \parencite{bruntonDataDrivenDynamicalSystems2019}}
    \label{fig:koopman_visualisation}
\end{figure}

The operator is however infinite in dimension. This provides a significant challange for computing. Instead of capturing
the development of all the measurement functions, one can identify key measurement functions that evolve linearly over time.
\parencite{bruntonDatadrivenScienceEngineering2019}. This can be done by determining the eigenfunctions of the koopman
operator \( (\varphi(\mathbf{x}_k)) \). 

\begin{equation}
    \varphi(\mathbf{x}_{k + 1}) = \mathcal{K}\varphi(\mathbf{x}_k) = \lambda \varphi(\mathbf{x}_k)
\end{equation}

These eigenfunctions allow one to globally model nonlinear dynamics with a linear representation. A set of Koopman
eigenfunctions can be used to generate more eigenfunctions. The product of two eigenfunctions is also an eigenfunction.

\begin{align}
    \mathcal{K}(\varphi_1(\mathbf{x})\varphi_2(\mathbf{x})) &= \varphi_1(\mathbf{F}(\mathbf{x}))\varphi_2(\mathbf{F}(\mathbf{x})) \\
    &=\,\lambda_1\lambda_2\varphi_1(\mathbf{x})\varphi_2(\mathbf{x})
\end{align}

This means there may be a finite set of eigenfunctions that can be used to construct all infinite eigenfunction needed
to fully represent the nonlinear system. 

\subsubsection{Finite dimensional models}

As the eigenfunctions form a basis for the measurement space, each measurement function can be formualated as an
infinite weighted sum of eigenfunctions.

\begin{equation}
    g_i(\mathbf{x}) = \sum_{j = 1}^{\infty} v_{ij}\varphi_j(\mathbf{x})
    \label{eq:koopman_measurement_spec_decomp}
\end{equation}

If we consider a measurement vector \(\mathbf{g}(\mathbf{x}) = [g_1(\mathbf{x}), \dots, g_p(\mathbf{x})]^T \),
\eqref{eq:koopman_measurement_spec_decomp} can be formulated in vector form:

\begin{equation}
    \mathbf{g}(\mathbf{x}) = \begin{bmatrix}
        g_1(\mathbf{x}) \\
        g_2(\mathbf{x}) \\
        \vdots         \\
        g_p(\mathbf{x})
    \end{bmatrix}
    = \sum_{j = 1}^{\infty} \varphi_j(\mathbf{x}) \mathbf{v}_j
\end{equation}
Where \(\mathbf{v}_j\) is the $\mathit{j}$-th Koopman mode associated with $\varphi_j$. For practical applications, this
sum is truncated to make an approximation of the measurement space. The challenge of using the Koopman operator has now
shifted to selecting the best finite set of eigenfunctions to capture the systems dynamics.

\subsubsection{Data-driven Koopman analysis}
Analytically Determining the Koopman eigenfunctions and their weights for a complex non-linear dynamical system is not
practically feasible. As such data-driven techniques are the method of choice for real world systems. 

DMD is a data driven technique used to identify modes that represent spatial patterns that evolve linearly in time. DMD
approximates the Koopman operator while restricted to a set of direct measurements of the state. It is based purely on
data, and requires no prior knowledge of the system's governing equations. Its simple and efficient numerical
implementation has made it popular in recent years \parencite{bruntonDataDrivenDynamicalSystems2019}. 

The governing principle is as follows: Snapshots of the dynamical system are arranged into two data matrices. These
matrices are then used to solve for a linear operator that maps one data matrix onto the other. 

\begin{equation}
    \label{eq:dmd_X_matrices}
    \begin{split}
    \mathbf{X} &= \left[\mathbf{x}(t_1), \mathbf{x}(t_2),\dots,\mathbf{x}(t_m) \right] \\
    \mathbf{X'} &= \left[\mathbf{x}(t'_1), \mathbf{x}(t'_2),\dots,\mathbf{x}(t'_m) \right]
\end{split}
\end{equation}

\begin{equation}
    \mathbf{X'} \approx \mathbf{AX}
\end{equation}

DMD provides a method for obtaining $\mathbf{A}$ efficiently using a singular value decompostion of $\mathbf{X}$.

\begin{equation}
    \mathbf{y} = \Theta^T(\mathbf{x}) = \begin{bmatrix}
        \theta_1(\mathbf{x})\\
        \theta_2(\mathbf{x})\\
        \vdots\\
        \theta_p(\mathbf{x})
    \end{bmatrix}
\end{equation}

This method can be extended by using a library of nonlinear measurement functions and fitting a linear operator to the
augmented state. Then two augmented data matrices are constructed, analogously to \eqref{eq:dmd_X_matrices}. These can then be solved to
obtain a linear operator. If $\Theta$ fully spans a Koopman invarient subspace, the extended DMD operator converges to
the Koopman operator in the limit of $m$. However, if $\Theta$ contains the wrong functions, the model will overfit or
give incorrect modal responses. Correctly determining the functions that should be contained in $\Theta$ is thus
essential.

\subsubsection{Neural networks for Koopman theory}
Even for relatively simple dynamical systems, the Koopman eigenfunctions can be arbitrarily complex
\parencite{bruntonDataDrivenDynamicalSystems2019}. Creating a suitable library for use with eDMD can be challenging.
Deep learning could provide an alternative approach \textcite{luschDeepLearningUniversal2018}

The goal of determining an invertable function \(\varphi\) and Koopman operator \(\mathcal{K}\) such that \( y_{k} =
\varphi(x_k) \) and \( y_{k + 1} = \mathcal{K}y_k \). This problem fits well with a deep auto-encoder based approach. An
additional constraint added is that the dynamics of the latent variables must linarly represent the dynamics of the
system. This forces the encoder to act as Koopman eigenfunctions.

    \label{fig:lusch_koopman_AE_architecture}

In the case where the system has continuous eigenvalues, the neural net representing \(\mathcal{K}\) can be parametrized
for the eigenvalue of the system. This can be obtained with another neural net on the latent variables.
\textcite{luschDeepLearningUniversal2018} indicate this approach still works on systems that have discrete Koopman
eigenvalues, meaning no prior knowledge is needed on the nature of the Koopman eigenvalues. 

This approach is not without its challenges. While there is no more need to select a basis for the eDMD method, one now
has to decide on the number of latent nodes needed, as well as other hyperparameters. This approach also has a larger
upfront computational cost to train the system as well as needing a large amount of data.

\subsubsection{Koopman theory for stochastic systems}
The above sections discuss the Koopman operator for deterministic systems. If a stochastic dynamical system is to be
considered however, the formulation can be adjusted. This is given by \textcite{crnjaric-zicKoopmanOperatorSpectrum2020}
in the following form: 

\begin{equation}
    \mathcal{K}g(\mathbf{x}) = \mathbb{E}[g(\mathbf{F}(\mathbf{x}))]
\end{equation}

\end{equation}

where $\varphi$ are the eigenfunctions of the stochastic Koopman Operator and $\lambda^S(t)$ are the stochastic Koopman

\subsection{Variational approach for Markov processes}
\textcite{wuVariationalApproachLearning2019} developed a method to approximate the Koopman operator from data of
stochastic systems. Their approach is based off stochastic koopman theory coupled with canonical covariance analysis. This
approach was extend by \textcite{mardtVAMPnetsDeepLearning2018} to use deep learning. 

The method uses two neural encoders to estimate the measurement function at any given timestep. These are indicated as
\(\mathcal{X}_0 \text{ and } \mathcal{X}_1\) respectively. By passing two states through these networks time lagged by
by time \(\tau\), 
the covariance can be calculated.

\begin{equation}
    \begin{aligned}
        C_{00} &= \mathbb{E}_t \left[\mathcal{X}_0(\mathbf{x}_t)\mathcal{X}_0(\mathbf{x}_t)^T\right] \\
        C_{01} &= \mathbb{E}_t \left[\mathcal{X}_0(\mathbf{x}_t)\mathcal{X}_1(\mathbf{x}_{t + \tau})^T\right] \\
        C_{11} &= \mathbb{E}_{t + \tau} \left[\mathcal{X}_1(\mathbf{x}_{t + \tau})\mathcal{X}_1(\mathbf{x}_{t + \tau})^T\right] \\
    \end{aligned}
\end{equation}

The optimal koopman operator can then be found by minimising the least square error between
\(\mathcal{X}_1(\mathbf{x}_{t + \tau})\) and \(\mathcal{X}_0(\mathbf{x}_t)\). This becomes:

\begin{equation}
    \mathcal{K} = C_{00}^{-1} C_{01}
\end{equation}

Directly solving this would result in a trivial answer. To avoid that issue,
\textcites{wuVariationalApproachLearning2019}{mardtVAMPnetsDeepLearning2018} use the VAMP-2 score. This additionally has
an advantage for deep learning, as the gradients are easier to handle, making the approach simpler for time series data.

\begin{equation}
    \label{eq:VAMP_2}
    \hat{R}_2\left[\mathcal{X}_0, \mathcal{X}_1\right] = \left|\left| C_{00}^\frac{-1}{2}C_{01} C_{11}^\frac{-1}{2}\right|\right|^2 
\end{equation}

By using \eqref{eq:VAMP_2} as a loss function for the two networks, these can be trained to learn \(\mathcal{X}_0 \text{ and } \mathcal{X}_1\).

\subsection{DeepONet}
DeepONet, proposed by \textcite{luLearningNonlinearOperators2021}, works based on the universal operator approximation
theorum. The theorum states that a neural network with a single hidden layer can approximate any nonlinear continuous
function or operator. 

\textcite{luLearningNonlinearOperators2021} used this theorum to develop DeepONet, a network architecture designed to
approximate nonlinear operators. Let \(u(x)\) be an input signal with \(G(u)\) is the corresponding output function.
Then \(G(u)(y)\) is a real number. The network therefore takes two inputs, the function \(u(x)\) and the point at which
to sample \(y\). 

The universal approximation theorum for operator is as follows \parencite{chenUniversalApproximationNonlinear1995}. Let \(\sigma\) be a continous non-polynomial function.
\(G\) is a nonlinear continuous operator. For any \(\epsilon > 0\) there exists a set of positive integers \(n,p,m\) and
constants \(c_i^k, \zeta_{ij}^k, \theta_i^k, \zeta_k, \zeta_k\in\mathbb{R}, x_j\in\mathbb{R}, w_k\in\mathbb{R}, i = 1,
\dots, n, k = 1,\dots,p, j = 1,\dots m \) such that:

\begin{equation}
    \left| G(u)(y) - \sum^p_{k = 1} \underbrace{ \sum^n_{i = 1} c_i^k\sigma\left(\sum^m_{j = 1}\zeta_{ij}^ku(x_j) + \theta^k_i\right)}_{\text{Branch network}} \, \underbrace{\sigma(w_k\cdot y + \zeta_k)}_{\text{Trunk network}}\right| < \epsilon
\end{equation}

\textcite{luLearningNonlinearOperators2021} adapts this formulation from using a single shallow network to two deep
networks: a branch network and a trunk network. These are then combined to form the total operator. The branch net can
consist of multiple networks and handels
the input function whereas the trunk network handles the location to evaluate the operator at. The specific network
architecture choices for the branch and trunk networks are arbitrary and depend on the problem at hand. 

\begin{equation}
    G(u)(y) \approx \sum^p_{k = 1}\underbrace{b_k(u(x_1), u(x_2), \dots, u(x_m))}_{\text{Branches}}\,\underbrace{t_k(y)}_{\text{Trunk}}
\end{equation}

In practice, \(p\) is often of at least order 10 \parencite{luLearningNonlinearOperators2021}. As such the branch
networks are combined into a single network that outputs a vector \([b_1, b_2, \dots b_p]\). This difference, as well as
the overall architecture is illustrated in \cref{fig:deeponet_architecture}. The architecture with a single network is
referred to as \emph{unstacked} whereas the version with separate branch networks is referred to as \emph{stacked}.

Neural Ordinary Differential Equations are a powerful tool for modelling dynamics of systems. Neural stochastic
differential equations are their stochastic analogue. This section will discuss these approaches for use in modelling
stochastic dynamical systems. First an overview will be given of neural ODEs, followed by an overview of neural
stochastic differential equations. 

\subsection{Neural Ordinary Differential Equations}
A second method for modeling dynamics is neural ordinary differential equations (Neural ODEs). First published by
\textcite{chenNeuralOrdinaryDifferential2018}, neural ODEs are derived from residual neural networks (ResNet). A ResNet
works as follows:

\begin{equation}
    \mathbf{h}_{k + 1} = \mathbf{h}_k + f_\theta\left(\mathbf{h}_k, k\right)
    \label{eq:resnet}
\end{equation}

These iterative updates can be interpreted as an Euler discretisation of a continuous system. In the continuous time limit, the dynamics can be described by the following:

\begin{equation}
    \diff{\mathbf{h}(t)}{t} = f_\theta(\mathbf{h}(t), t)
    \label{eq:neural_ode}
\end{equation}

The time stepping can then be handled by a time integration algorithm. 

\subsubsection{Advantages of Neural ODEs}
Neural ODEs have several advantages over ResNets
for the modelling of dynamical systems \parencite{chenNeuralOrdinaryDifferential2018}. 

ResNets approximate the Euler discretisation of a dynamic system. This is one of the simplest time integration methods
and numerous more efficient and accurate integration schemes are available. Neural ODEs allow the use of more advanced time stepping schemes
allowing the cost of the model to scale better with the complexity of the problem. Furthermore, as neural ODEs act in
continuous time, the effect of irregularly sampled data mitigated. When training on simulated data, this is less
significant, but when using real data from various sensors on board a ship this can be a significant advantage. 

A further advantage is that neural ODEs do not calculate the gradients of the weights using backpropagation. Rather, a second ODE
is solved backwards in time and the gradients are obtained from there. This means that the memory cost of training is
independent of the length of the time series. 

Finally, for the modelling of time series, a neural ODE, by its nature, uses the same parameters for each timestep. This
reduces the complexity of the model as well as improving the generalizability of the network.

\subsubsection{Limitations of Neural ODEs}
These advantages are not without their limitations however. Many of the limitations of solving regular ODEs are also
prevelant with neural ODEs. The computational cost and accuracy is highly dependent on the solver chosen. For stiff
systems explicit solvers become infeasible so implicit solvers are needed, increasing computational cost
\parencite{anantharamanStablyAcceleratingStiff2022}. Neural ODEs are also difficult to batch.
\textcite{chenNeuralOrdinaryDifferential2018} states that this would be possible by concatenating the states of each
batch, though this might require a large number of system evaluations extra compared to solving individually. Neural
ODEs must also be Lipschitz continuous to have a solution. This holds if the weights of the network are finite and the
activation functions are also Lipschitz continuous. 

Due to the sequential nature of neural ODEs, it is challenging to run them parallel on modern hardware such as GPUs.

\subsection{Theoretical formulation of neural SDEs}
\label{subsec:neural_sde}
A limitation primarily relevant to this study is that in the form of neural ODEs given in \eqref{eq:neural_ode}, the network is unable
to deal with stochastic systems. Stochastic differential equations (SDEs) can similarily be approximated by neural
networks to give a neural SDE. The general form of these is given as follows \parencites{jiaNeuralJumpStochastic2020}{liuHowDoesNoise2020}{tzenNeuralStochasticDifferential2019}:

\begin{equation}
    dh(t) = f_\theta(h(t), t) dt + g_\varphi(h(t), t) dW, \quad dW \sim \mathcal{N}(0, dt)
\end{equation}

Where \(f\) and \(g\) are neural networks parameterised by weights \(\theta\) and \(\varphi\) respectively. \(W\) is a
wiener process sampled from a gaussian distribution. \(f\) is the drift term, this describes the deterministic dynamics
of the system. The diffusion term, \(g\), models the stochastic components of the dynamical system. 

Similar to neural ODEs, this formulation results in a model that effectively defines the dynamics of the system in
continuous time. To progress the model in time, a stochastic time integration scheme can be used such as Euler-Maruyama,
Stochastic Runge-Kutta or stochastic Heun integration \parencite{kidgerEfficientAccurateGradients2021}.

\subsection{Adjoint method for neural SDEs}
\label{subsec:adjoint_sde}

Training neural SDEs and ODEs ideally avoids storing the full forward path in GPU memory, but this conflicts with
backpropagation requirements, which depend on intermediate states to compute gradients with respect to model parameters.
To work around this limitation in neural ODEs, the adjoint sensitivity method is typically used: instead of
backpropagating through every solver step, one solves an adjoint ODE backwards in
time.\parencite{chenNeuralOrdinaryDifferential2018} \textcite{liScalableGradientsStochastic2020} adapted this idea to
the stochastic setting, showing how to define and numerically solve a backward-in-time adjoint \emph{SDE} that yields
gradients with constant memory cost.

Consider a Stratonovich SDE with Lipschitz continuous drift \(b\) and diffusion \(\sigma\)

\[
Z_T = z_0 + \int_0^T b(Z_t, t)\,dt + \int_0^T \sigma(Z_t, t) \circ dW_t,
\]

and write \(\Phi_{s,t}(z) := Z_t^{s,z}\) for the solution at time \(t\) started from state \(z\) at time \(s\). For fixed \(s\) and \(t\), we can also view \(\Phi_{s,t}\) as a (random) change of variables on state space, with
inverse map

\[
\check{\Psi}_{s,t} := \Phi_{s,t}^{-1}.
\]

\(\check{\Psi}_{s,t}(z)\) can be understood as: "which state at time \(s\) would have evolved into \(z\) at
time \(t\), under this particular realization of the Brownian path?". \textcite{liScalableGradientsStochastic2020} show
that \(\check{\Psi}_{s,t}\) itself satisfies a backward Stratonovich SDE. This backward SDE
simply describes running the original dynamics in reverse time along the same stochastic path, with appropriate sign
changes in the drift and diffusion terms.

The goal of the adjoint method is to compute the gradient of a scalar loss \(\mathcal{L}\) that depends on the terminal
state \(Z_T\), with respect to both the initial condition and the model parameters (which enter through \(b\) and
\(\sigma\)). We define the adjoint process by

\[
A_{s,t}(z) := \frac{\partial \mathcal{L}(\Phi_{s,t}(z))}{\partial z},
\]

so that \(A_{s,t}(z)\) represents the sensitivity of the loss to perturbations of the state at time \(t\), when the
system was started from \(z\) at time \(s\). In particular, \(A_{0,T}(z_0)\) is the gradient of the loss with respect to
the terminal state produced by the forward SDE. 

Crucially, \textcite{liScalableGradientsStochastic2020} show that this adjoint process can be characterized as the
solution of a backward SDE that is coupled to the original dynamics: as we integrate backwards from \(T\) to
\(0\), we jointly evolve the state via the backward flow \(\check{\Psi}\), and the adjoint via an SDE whose
drift and diffusion depend on the Jacobians of \(b\) and \(\sigma\) with respect to the state and parameters. Instead of
storing the entire forward trajectory and differentiating through the solver, the
dynamics can be re-simulated in reverse time and propagate sensitivities through this backward SDE. This yields
gradients with respect to initial conditions and parameters using only \(\mathcal{O}(1)\) memory, at the cost of solving
an additional SDE backwards in time.

The adjoint SDE is not solved analytically but numerically, using essentially the same solver infrastructure as for the
forward dynamics. Following \textcite{liScalableGradientsStochastic2020}, the backward integration is done with a chosen
timestep such that the adjoint variables and the backward flow are jointly evolved from $T$ back to~$0$. This timestep
does not have to be the same as that of the forward SDE. This mirrors the continuous-time adjoint method for ODEs, but
now in the stochastic setting.

A challenge is that the backward SDE must reuse the identical Brownian path from the forward computation, while
still avoiding storage of all forward-time increments. \textcite{liScalableGradientsStochastic2020} resolve this using
the virtual Brownian tree (VBT), which reconstructs increments on demand via Brownian bridges and splittable
random seeds, achieving $\mathcal{O}(1)$ memory and logarithmic-time generation for any interval. This enables adaptive
step sizes in the forward solver without memory overhead. This method was improved upon by
\textcite{kidgerEfficientAccurateGradients2021}. They introduce the Brownian Interval construction. Unlike VBT,
which reconstructs increments in $\mathcal{O}(\log L)$ time and with small approximation errors, the Brownian Interval
provides exact Brownian samples with average $\mathcal{O}(1)$ time and memory cost, using a binary interval tree together with a
small LRU cache. This yields substantial speedups and improves stability of the backward SDE by avoiding stochastic noise from approximate reconstruction.

From a numerical-analysis perspective, the backward SDE is expressed in Stratonovich form. Gradients with respect to
parameters are accumulated using vector-Jacobian products at each backward step, keeping memory constant while
remaining compatible with automatic differentiation. \textcite{kidgerEfficientAccurateGradients2021} propose an
algebraically reversible solver for Stratonovich SDEs: the reversible Heun method. It allows exact
reconstruction of all intermediate forward states during the backward pass, eliminating the gradient inaccuracies that
arise when standard solvers must be (approximately) inverted in reverse-time adjoint integration. Its computational cost
remains low—one function evaluation per step—while matching the strong convergence rates of classical midpoint/Heun schemes.

Literature discusses two main training paradigms for neural SDEs. These are the \emph{Latent SDE} method proposed by
\textcite{liScalableGradientsStochastic2020} and the \emph{SDE-GAN} approach by
\textcite{kidgerNeuralSDEsInfiniteDimensional2021}. 

Both approaches model the dynamics in a latent space because the observations themselves are not Markovian \parencite{kidgerNeuralSDEsInfiniteDimensional2021}. In practice,
observations are partial, noisy, and often irregularly sampled, so they do not contain all information required to
predict future behavior. This means each observation still depends on earlier (unobserved) states, which makes the
observed process non-Markovian. Latent-space representations are commonly used for this reason: they compress the data
into a form where essential features are retained while removing distortions from the observation process. 

In contrast, the latent dynamics can be modeled as Markovian, since the latent state is constructed to contain all
information needed for future evolution. Neural latent-SDE and related frameworks explicitly exploit this by learning
hidden dynamics that depend only on the current latent state, rather than on the full observation history. This allows
efficient forecasting and simulation without storing past states.

\subsubsection{Latent SDE}
The latent SDE method was developed by \textcite{liScalableGradientsStochastic2020} to train neural SDEs. The key
problem it aims to avoid is the fact that directly fitting an SDE to observed data by maximising the likelyhood often
causes the learned diffusion to tend to zero. This causes the model to overfit the data. It overcomes this issue by
using a variational approach. 

The approach is similar to that used in a variational autoencoder \parencite{kingmaAutoEncodingVariationalBayes2014}. A
prior and approximate posterior are assumed as Neural SDEs, each with their own drift term but a shared diffusion term.

\begin{equation}
    \begin{aligned}
        d\tilde{z}_{t}&=\mu_{\theta} (\tilde{z}_{t},t)dt+\sigma (\tilde{z}_{t},t)dw_{t}& ({{\mathrm {prior}}})\\ 
        dz_{t}&=\mu _{\phi} (z_{t},t)dt+\sigma (z_{t},t)dw_{t} &({{\mathrm {approx. ~posterior}}})\\
        \tilde{z}_{0}&=z_{0}\sim \psi (x_{0})& ({{\mathrm {initial~ latent ~state}}})\\ 
        x_{t}& = f(z_{t})& ({{\mathrm {decoded ~states}}})
    \end{aligned}
\end{equation}
The drift and diffusion functions, \(\mu_{\theta}, \mu _{\phi} \text{ and } \sigma \) are Lipschitz in state and time.
\( \psi \text{ and } f\) are an encoder and decoder respectively. 

The approximate posterior aims to learn the true posterior distribution of the system. It does this by mapping
observations to a latent path. The posterior network does this with information from the observations.
\textcite{liScalableGradientsStochastic2020} achieved this by encoding observations into a context vector which is
passed to the drift network each timestep.

The prior aims to generate latent trajectories that match the approximate posterior. It generates a latent path from a
given starting point. Unlike with a variational autoencoder, the prior is not fixed and is learned as it needs to capture the latent dynamics.

The decoder network encodes the latent space states into observation space. The encoder network can be used in two ways.
\textcite{ryzhikovLatentStochasticDifferential2023} used it to encode the initial state of the observations to determine
the initial latent state. \textcite{liScalableGradientsStochastic2020} used a gated recurrent unit (GRU) encoder to add a context vector to the
input parameters of the the approximate posterior.

The loss function is the ELBO. It consists of two components, the reconstruction loss and the KL divergence loss. The
reconstruction loss minimises the difference between the observations and the decoded latent state generated by the
posterior SDE. The KL divergence loss maps the difference between the paths generated by
the prior and the paths generated by the posterior. It can be written as:

\begin{equation} 
    \log p(x_{1}, x_{2}, {\dots }, x_{N}|\theta) \geq \mathbb{E}_{z_{t}}\left[{\Sigma
_{i=1}^{N}\log p(x_{t_{i}}|z_{t_{i}})-\int _{0}^{T}\frac {1}{2}|u(z_{t},t)|^{2}dt}\right]
\end{equation}

Where:

\begin{equation} 
    u(z_{t},t)=\frac {\mu _{\theta} (z_{t},t)-\mu _{\phi} (z_{t},t)}{\sigma (z_{t},t)}
\end{equation} 

The term containing \(u(z_t, t)\) can be considered as the KL-divergence term. By augmenting the foreward SDE's state
with an additional variable with drift \(\frac {1}{2}|u(z_{t},t)|^{2}\) and diffusion \(0\), the gradient with respect
to the prior and variational parameters can be determined using the adjoint method discussed in \cref{subsec:adjoint_sde}.

Overall in the training scheme, the posterior enusures the true dynamics are captured in the latent space. The prior is
trained to generate latent dynamics and the decoder converts latent space into observed space. The latent SDE is
relatively easy to train, however similar to variational inference, the diffusion will often be too small \parencite{kidgerNeuralDifferentialEquations2022}

\subsubsection{SDE-GAN}
The SDE-GAN method trains the neural SDE as a generative adversorial network (GAN). The SDE-GAN method was proposed by
\textcite{kidgerNeuralSDEsInfiniteDimensional2021}. The approach stems from traditional SDE training methods where some
statistics of the model are fit to the data. 

A GAN works by training two networks, a generator and a discriminator. The generator creates a dataset from noise; the
discriminator aims to distinguish the generated dataset from a real dataset. By pitting these two networks against each
other, the generator eventually learns to produce realistic datasets.

\textcite{kidgerNeuralSDEsInfiniteDimensional2021} train neural SDEs through such a framework. The generator is the
neural SDE to be trained. This is passed a randomly sampled initial state and brownian motion and parametrised by \(\theta\).

\begin{equation}
    \begin{aligned}
        dz_{t}&=\mu _{\theta} (z_{t},t)dt+\sigma_\theta (z_{t},t)dw_{t}\\
        z_{0}&=\psi_\theta(v) \quad v\sim \mathcal{N}(0, 1)\\ 
        x_{t}& = f_\theta(z_{t})
    \end{aligned}
\end{equation}

The discriminator is another neural SDE, paramtrised by \(\phi\). 

\begin{equation}
    \begin{aligned}
        dh_{t}&= f_{\phi} (h_{t},t)dt+ g_\phi (h_{t},t)dx_{t}\\
        h_{0}&=\psi_\phi(x_0) \\ 
        x_{t}& = m_\phi(z_{t})
    \end{aligned}
\end{equation}

Rather than relying on a brownian motion path, the discriminator uses the generated or observed path. This means the
discriminator can be viewed as a neural controlled differential equation.

    \centering
    \includegraphics[width=\linewidth]{report/images/literature_study/kidgerNeuralSDEsInfiniteDimensional2021.png}
    \caption{Visualisation of SDE-GAN \parencite{kidgerNeuralSDEsInfiniteDimensional2021}}
    \label{fig:sde-gan_visualisation}
\end{figure}
The training loss used is the Wasserstein distance. The discriminator aims to maximise the difference between the
discriminated generated series and the discriminated data, whereas the generator aims to minimise the discretised score
of the generated data. 

\textcite{kidgerNeuralDifferentialEquations2022} state that while SDE-GANs generally produce more accurate results that
the latent SDE approach, it is more challenging to train. This is analogous to standard GAN training compared to VAE
training. GANs generally suffer from three major limitations: vanishing gradients, non-convergence and mode collapse \parencite{barshaIndepthReviewAnalysis2025}.
Vanishing gradients occur when the discriminator learns faster than the generator and becomes too profficient in
discrimitating. This results in the generator's gradient being negligable and preventing learning. Non-convergence is
when model parameters exhibit oscillatory behaviour, resulting in a failure to reach a converged state. Finally mode
collapse is when the generated outputs are restricted to a few specific cases rather than capturing the full diversity
of the training data. This is essentially overfitting of the model. 

A hybrid approach is briefly mentioned to avoid the disadvantages of both. This alternates which method is
used each training epoch.

\subsubsection{Futher work}
Futher research has been done on these methods. \textcite{el-lahamVariationalNeuralStochastic2025} and
\textcite{ryzhikovLatentStochasticDifferential2023} used the latent-SDE framework together with change point detection.
Multiple SDEs were trained with the latent SDE approach and switched between based off some change point metric. This
allows for the modelling of systems which have a sudden shift in the underlying dynamics. 

\textcite{liuContinuousTimeStochasticDifferential2021} introduces Variational Stochastic Differential Networks (VSDN), a
continuous-time stochastic recurrent model designed for irregular and sparse time-series data. VSDN embeds system
dynamics into latent neural SDEs whose drift and diffusion functions depend on latent states and historical observations
extracted through ODE-RNN encoders, enabling the model to capture complex stochastic dependencies. The work further
develops continuous-time VAE and IWAE objectives and provides both filtering and smoothing inference models, allowing
the posterior dynamics to incorporate past-only or past-and-future observations. Experiments on human motion-capture
datasets show that VSDN substantially outperforms existing continuous-time baselines, including LatentODE, GRU-ODE, and
LatentSDE, in both prediction and interpolation tasks.

Neural SDEs have also been used as regularisation for neural ODEs \parencite{liuHowDoesNoise2020}. This approach used a
fixed diffusion matrix, the form of which influencing which regularisation method was applied. The approach saw success
in increasing the robustness of neural ODEs

\textcite{lookDeterministicApproximationNeural2023} propose an alternative to Monte-Carlo-based inference for Neural
SDEs, which can be computationally expensive and produce poorly calibrated uncertainty estimates. Rather than relying on
sampling, they introduce a deterministic transition-kernel approximation called Bidimensional Moment Matching (BMM).
This method propagates the mean and covariance of the system state both through time and through the neural network
layers, allowing for uncertainty quantification without Monte Carlo sampling. The approach is designed to be
computationally efficient and scalable to high-dimensional systems while maintaining accurate uncertainty calibration.
The paper discusses the use of this method during testing, as opposed to sampling multiple times. Its ability to be
implemented during training is mentioned only in passing. 

\textcite{lookDifferentialBayesianNeural2019} developed a neural SDE using Bayesian neural networks as the drift and
diffusion networks. This allows their approach to capture both epistemic and aleatoric uncertainty. While the paper does
show their approach is better suited to capturing uncertainty than \parencite{liuHowDoesNoise2020}, it does not discuss
how the networks are trained.
